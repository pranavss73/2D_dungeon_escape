================================================================================
1. PROJECT OVERVIEW
================================================================================

  PROJECT NAME:    Dungeon Escape
  COURSE:          Data Structures II — Course Project
  ENGINE:          Custom C++ game engine built on SFML 2.x
  PLATFORM:        Windows (Visual Studio 2022, x64)
  GENRE:           Top-down 2D dungeon-crawler action RPG

  CORE CONCEPT:
  The player navigates a dungeon, fights waves of enemies using melee combat,
  solves lever-based puzzles to unlock new areas, collects a key to open the
  exit door, and escapes. Features persistent progression via a Skill Tree
  upgrade system and match history tracking.

  DSA FOCUS:
  • QuadTree — Spatial partitioning for efficient collision detection
  • N-ary Tree — Skill tree for persistent player upgrades
  • Vectors — Dynamic entity management throughout

6. ALGORITHM COMPLEXITY ANALYSIS
================================================================================

  ┌───────────────────────────────────────────────────────────────────────┐
  │  ALGORITHM                  │ COMPLEXITY      │ PER-FRAME CALLS     │
  ├───────────────────────────────────────────────────────────────────────┤
  │  QuadTree Build             │ O(n log n)      │ 1                   │
  │  QuadTree Query (per obj)   │ O(log n + k)    │ b (num bullets)     │
  │  QuadTree Insert (per obj)  │ O(log n)        │ n (num enemies)     │
  │  Player-Obstacle Collision  │ O(m)            │ 1 (m = obstacles)   │
  │  Enemy-Obstacle Collision   │ O(m)            │ n (per enemy)       │
  │  Melee AoE Hit Detection    │ O(n)            │ On attack only      │
  │  Enemy-Player Damage Check  │ O(n)            │ 1                   │
  │  Safe Spawn Validation      │ O(50 × m)       │ On spawn only       │
  │  Skill Tree Traversal       │ O(d)            │ On upgrade only     │
  │  CSV Parse (load records)   │ O(r)            │ On startup only     │
  └───────────────────────────────────────────────────────────────────────┘
  Where: n = enemies, m = obstacles, b = bullets, k = candidates, d = tree depth, r = records

9. BUILD & RUN INSTRUCTIONS
================================================================================

  PREREQUISITES:
  • Visual Studio 2022 (with C++ Desktop Development workload)
  • SFML 2.x libraries (included in dependencies/ folder)

  BUILD:
  1. Open project2.sln in Visual Studio 2022
  2. Set configuration to Debug | x64
  3. Build → Build Solution (Ctrl+Shift+B)

  RUN:
  1. Ensure SFML DLLs are in the output directory (x64/Debug/)
  2. Ensure assets/ folder is accessible from the working directory
  3. Press F5 (Debug) or Ctrl+F5 (Run without debugging)

  CONTROLS:
    W/A/S/D   — Move player
    SPACE     — Melee attack (costs 10 stamina)
    E         — Interact (levers, door)
    ESC       — Pause/Resume
    MOUSE     — Menu navigation (click buttons)

10. PERSISTENCE FILES
================================================================================

  save_data.txt — Match history (appended after each game):
    FORMAT: levelName,status,score,kills,coins
    EXAMPLE:
      Easy,Win,120,12,180
      Hard,Lose,50,5,75
      Hell,Win,340,34,430

  profile.txt — Persistent player profile:
    FORMAT (5 lines):
      Line 1: availableCoins (integer)
      Line 2: speed2 unlocked (0 or 1)
      Line 3: speed3 unlocked (0 or 1)
      Line 4: power2 unlocked (0 or 1)
      Line 5: power3 unlocked (0 or 1)
    EXAMPLE:
      3500
      1
      0
      1
      1
